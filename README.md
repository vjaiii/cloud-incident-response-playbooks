# Cloud Incident Response Playbooks

This repository documents practical incident response playbooks for common cloud security scenarios across modern environments.

The focus is on structured response thinking for cloud platforms, IAM events, Kubernetes incidents, suspicious activity, and data exposure risks.

---

## Why this repository exists

Cloud incidents move fast.

Security teams often need quick, repeatable guidance for:

- suspicious IAM activity
- exposed storage or data services
- unusual API behavior
- container or Kubernetes compromise
- unauthorized infrastructure changes
- logging and forensic review

This repository captures simple response playbooks that help teams investigate, contain, and recover in a structured way.

---

## Topics covered

- IAM compromise response
- Public storage exposure
- Suspicious service account activity
- Kubernetes incident triage
- Audit log investigation
- Containment and recovery steps
- Incident documentation guidance

---

## Example scenarios

Compromised cloud credentials

Unexpected privilege escalation

Public bucket or storage exposure

Suspicious workload behavior in Kubernetes

Unauthorized Terraform or infrastructure drift
